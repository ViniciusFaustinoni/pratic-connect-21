

# Fix: Duplicate `is_default_autentique` Flag and Legacy Contract Text

## Root Cause Analysis

After exhaustive investigation across all code, templates, and database content:

1. **"Serviços: {{plano.descricao}}" does NOT exist** in any current template, edge function code, logo image, or database record. The document in the screenshot was generated by an **older version** of the code, before the cleanup logic was deployed. Autentique PDFs are immutable — once generated, they cannot change.

2. **Critical bug found**: There are **two templates** with `is_default_autentique = true`:
   - `AF1` (Proposta de Filiação)  
   - `CONTRATO_ADESAO_V1` (Contrato de Adesão)
   
   The `.single()` query in `autentique-create` **fails silently** when two rows match, causing a fallback to the old hardcoded template (`generateTermoAfiliacao`). This means the database template is **never actually used** for new contracts.

## Changes

### 1. Database Migration — Fix duplicate default flag

Set only `AF1` as the default Autentique template (the "Proposta de Filiação" which is the active contract template). Remove the flag from `CONTRATO_ADESAO_V1`.

```sql
UPDATE documento_templates 
SET is_default_autentique = false 
WHERE codigo = 'CONTRATO_ADESAO_V1';
```

### 2. `supabase/functions/autentique-create/index.ts` — Add resilient query

Replace the `.single()` query with `.limit(1)` + manual extraction so it doesn't fail when multiple templates are flagged. Add a warning log if more than one template is found.

### 3. `supabase/functions/autentique-create-by-token/index.ts` — Same fix

Apply the same resilient query pattern.

### 4. Verification

After deployment, new contracts will use the database template (`AF1`) instead of the hardcoded fallback. The "Serviços: {{plano.descricao}}" text will not appear because:
- The `AF1` template does not contain it
- The cleanup regex in `gerarHTMLDoTemplate` still runs as a safety net
- `limparVariaveisNaoSubstituidas` catches any remaining `{{...}}` tags

Old documents already in Autentique cannot be changed — they would need to be regenerated if needed.
